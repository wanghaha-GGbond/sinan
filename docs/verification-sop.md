# 公司身份认证审核 SOP

## 适用范围

本流程适用于企业邮箱域名不匹配的人工复核，以及任职证明认证。审核只确认申请人与企业之间的任职关系，不授予企业查看匿名作者身份的权限。

## 审核步骤

1. 核对申请账号、姓名、职务、公司和联系邮箱是否一致。
2. 企业邮箱申请优先核对公司官网、公开招聘页或可信企业信息源中的域名。
3. 任职证明可接受在职证明、工牌或劳动关系材料；仅检查姓名、企业主体、在职状态和必要日期。
4. 要求申请人遮盖身份证号、住址、薪资、银行账户、员工编号等无关信息。
5. 材料存在明显编辑痕迹、主体不一致或无法确认时拒绝，并填写可操作的原因。
6. 审核通过后只保留审核结论、审核人、时间和授予的可信度，不保留原始证明文件。

## 判定标准

- 企业邮箱：域名可被可信公开来源确认属于目标公司，且申请人能够接收验证邮件。
- 任职证明：企业名称与申请公司一致，申请人姓名可辨认，材料能证明当前或近期任职关系。
- 域名属于集团或子公司时，需在审核备注中记录主体关系。
- 外包、供应商或客户邮箱不能作为目标公司的企业邮箱认证依据。

## 拒绝原因模板

- 邮箱域名与申请公司无可验证关联。
- 材料中的企业主体与申请公司不一致。
- 材料无法确认申请人的任职关系。
- 材料包含过多敏感信息，请遮盖后重新提交。
- 材料疑似经过影响真实性判断的编辑。

## 数据最小化

- 不收集身份证完整号码、薪资、住址、银行账户等信息。
- 原始材料审核完成后立即删除。
- 审核日志仅保留申请 ID、结论、原因、审核人和时间。
- 对外页面不得展示申请人的工作邮箱、真实姓名或证明材料。
