## 文档约定

- **P0 / P1 / P2**：P0 = 核心循环四件套（F1-F4），P1/P2 见 01 号文档排队表。
- **L1 / L2 / L3**：验证等级（企业邮箱 / 在职证明 / 薪资区间）。仅内部用语，**产品界面永不出现**。
- **trustLevel**：`users.trust_level` 字段，0=未验证，1/2/3 对应 L1/L2/L3。
- 涉及代码的描述以仓库现状为准：`apps/web/src/db/schema/` 为数据层事实源。
- 每份 spec 末尾有「验收标准」，与 09 号文档的指标表一致；冲突时以 09 为准。

## 核心主线（一句话版）

**身份卡（资产）→ 部门级评价 + 情绪指数（内容护城河）→ 邀请制（增长杠杆）→ 拍卖（变现与传播引爆点）。**

预测市场、可视化公司放后期：前者有监管风险需要包装，后者是体验加分项不是命脉。
