# 09 · 指标体系

> 北极星只有一个；守门指标防止增长压垮信任；漏斗用于定位问题。各 spec 验收线与本表冲突时以本表为准。

## 1. 北极星

**L2 以上验证用户的周活跃率**（验证身份 × 活跃 = 本产品独有且竞品无法复制的资产）。

口径：当周有任意主动行为（发布/投票/讨论/邀请/竞拍）的 trustLevel≥2 用户 ÷ 全部 trustLevel≥2 用户。

## 2. 守门指标（反向约束，与北极星同屏展示）

| 指标 | 阈值 | 触发动作 |
|---|---|---|
| 评价被申诉成功率 | >3% | 审核质量复盘，暂缓拉新投放 |
| 删帖率（非作者自删） | >5% | 同上 |
| 验证吊销率 | >1% | 验证审核 SOP 复盘（02 §8） |
| 单公司负面密度异常 | 聚类告警 | 反作弊流程（08 §6） |

逻辑：北极星涨而守门破线 = 假繁荣，按守门指标行动。

## 3. 漏斗

### 增长漏斗（F3）
```
落地页曝光 → 注册（>53%）→ L1 验证（>60%，被邀请人口径）→ 首次发布 → 发出首个邀请
K 因子 = 人均发出邀请 × 注册转化，目标 >0.8（推导见 04 §1.4）
```

### 身份漏斗（F1）
```
注册 → 任意等级验证 >60% ；注册 → L1 >40% ；验证用户 → L2 >25%
```
（v0.1 的「注册→L1 >60%」修订为 40%：企业邮箱收码有留痕顾虑，详见 02 §3；60% 移到「任意等级」口径。）

### 内容漏斗（F2）
```
L2 用户 → 30 天内发布首条评价 >40% → 评价获 useful >60% → 发布第二条 >50%
```

## 4. 里程碑验收线（汇总）

| 阶段 | 验收 |
|---|---|
| M0 | 种子 500 人；注册→任意验证 >60%；邀请使用率 >50% |
| M1 | 50 家公司；Top 50 每家 ≥3 部门 ≥5 条 L2 评价；DAU/MAU >25%；守门指标全部达标 |
| M2 | MAU 2 万；K >0.8；媒体报道 ≥3 篇；拍卖 10 场、场均 >20 人、≥1 次破圈 |
| M3 | 次月留存 >40%；拍卖履约 >95%、纠纷 <2%；首批 3 圈各 ≥50 人 |
| M4 | B 端首单或积分竞猜内测留存数据（探索期，不设硬线） |

## 5. 冷启动密度纪律

前 6 个月**只做 20 家公司**：单公司渗透率（站内验证用户 ÷ 估算员工数）>5% 才允许扩下一批。
宁可 20 家公司每家 100 人，不要 200 家公司每家 10 人——部门级评价 + k-匿名阈值（≥5 人才展示）决定了**密度不够时产品根本不成立**。

## 6. 实现（M2 · S10）

- 第一版 = SQL 聚合 + admin 看板页，不接 BI（07 不做清单）。
- 事件埋点最小集：注册（带邀请码）、验证各级达成、发布（类型）、useful、邀请发出/使用、分享卡生成、落地页 UV。
- 周报自动化：北极星 + 守门四项 + 三漏斗，每周一生成。
